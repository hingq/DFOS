# Output

## 文件

| 文件 | 说明 |
|------|------|
| `data/output/{DATE}-article.md` | **对话式结构化长文**，且为 **TTS 唯一母本** |

## 体裁建议（对话式结构）

以下为结构示例，具体以 `briefing-article-style` skill 为准：

```markdown
# 标题：简洁有力

**角色A**：…（身份一句）
**角色B**：…（身份一句）

---

## 开场
**角色A**：…
**角色B**：…

---

## 主题一：…
（多轮对话，有观点碰撞）

---

## 主题二：…

---

## 编辑收束 / 结尾
**角色A**：…
**角色B**：…
```

也可采用「主持 + 分析师」或分节小标题 + Q&A，但须保持 **同一文件** 内完成可播讲与可发布。

## 与旧版 article-writing 的说明

- 不再强制单独的 `Key Points / Background / Details` 纯分段式结构；若 skill 要求保留其中部分要素，可嵌入对话节内。  
- **不再**将 `*-dialogue.md` 作为与 `article` 并列的第二正文。

## 验收检查

- [ ] 单文件 `article.md` 即对话式成稿  
- [ ] 覆盖当日核心资讯（与 processing 一致）  
- [ ] 有编辑判断或收束段  
- [ ] 可作为 TTS 直接输入（角色标注清晰）
