# 反馈回路

## 记录层

将以下内容写入 `memory.md` 或项目日志：

- **搜索关键词效果**：哪些关键词找到了高质量情报？
- **漏判情况**：应该发现但未发现的KOL或情报
- **误判情况**：误认为重要但实际价值低的内容
- **用户偏好**：用户对情报详细程度、领域偏好的调整
- **粉丝数抓取**：autocli（或所用命令）对哪些账号稳定返回粉丝数、哪些经常失败或缺失，便于调整拉取节奏或提出 Skill 候选

## 项目层

仅当规则在本项目中稳定复现时，写入项目规则文件（如 `rules.md`）：

- 高质量KOL识别标准
- 实体词提取优先级规则
- 去重判定阈值

## Skill候选层

以下情况可提出 Skill 更新候选：

- `autocli` 搜索语法优化（如验证有效的新搜索模式）
- 多语言内容处理（跨语言情报收集）
- **Twitter 资料字段**：若某子命令能稳定暴露粉丝数等字段，可整理为候选（附验证证据）

提出时需包含：
- 候选规则内容
- 验证证据（至少3个成功案例）
- 适用范围说明

## 约束

- 默认不自动修改 `.claude/skills/**`
- 一次性异常不沉淀为规则
- Skill 升级需人工审阅
