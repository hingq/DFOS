# 约束

## 与 Daily Briefing 的关系（简报素材合流）

- Twitter 采集与 **其他信源爬取** **共同**作为生成 `briefing.md` 的资源；`latest.md`、KOL 清单等是**工作记录与溯源**，用于支撑合流与运营，**不是**与简报并列的独立「日报产品线」。  
- 工程上常在运行 `pipeline.main` 前完成 Twitter 采集并写入 raw 或辅助路径；若当日 Twitter 抓取失败，应记录并重试或按规程处理，避免简报在设计上缺失该渠道素材。

## 数据与诚信

- **不得编造**：推文 ID、用户名、发布时间、正文、粉丝数等；未取得可靠值则不写、不估。  
- **粉丝数**：仅作传播面参考；汇总表排序不等于重要性排序。

## 工具与边界

- 主要工具：`autocli` 等（见 background）。  
- 不访问非授权或非公开信息。
