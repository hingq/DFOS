# Goal

## 交付成果

每天产出一篇可直接发布的简报 Markdown，存储到 `data/output/YYYY-MM-DD-briefing.md`。

## 简报结构

1. **今日必看**（2-3条，score ≥ 4.0）：完整报道 + 编辑视角
2. **值得关注**（4-6条，score 3.0-3.9）：简明扼要
3. **工具雷达**（0-3条，score 2.0-2.9 且 category=new_tool）：一行提及
4. **编辑视角**（0-1条）：产品架构师角度的深度分析

## 验收标准

- 总条数 8-12 条
- 每条附来源链接
- 编辑视角关联 context 文件中的判断框架
- 写作风格符合 context/style.md
- 全文可在 5 分钟内读完
